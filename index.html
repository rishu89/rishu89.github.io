A secure task-offloading framework for fog computing environment

B.tech Project

I always had a fascination of what exactly blockchain is and how it works. As I have already worked on Fog computing architecture and was aware of the the most important challenges that is present in the architecture which is how fog devices is going to trust other fog devices. Then there comes the idea of combining the fog architecture with blockchain environment. This project presents a secure task offloading framework using blockchain technology. The proposed framework brings in transparency, and security into a distributed and trust-less environment like fog-computing, where multiple IoT applications can depend on the peer fog-nodes to meet their QoS requirements. The proposed mechanism considers a permission-less blockchain that allows easier integration of different IoT applications into fog network. To prevent unauthorized access to the fog, we use the public-key infrastructure to verify the authenticity of nodes with lower reputation/rank.The proposed scheme allows nodes to verify a job trail, in case of data misuse.

BDmark: A Blockchain-Driven Approach to Big Data Watermarking

Research project || Published in Springer conference

A novel approach BDmark that provides a transparent immutable audit trail for data movement in big data monetizing scenarios, by exploiting both watermarking and blockchain technologies power. We considered a complex big data monetization scenario where multiple actors were involved in data collection, aggregation, storage, selling, and purchasing. An access control mechanism to ensures that only legitimate buyers, with data owner's permission,can get access to the target data-piece.I have created a model using solidity and java language and developed an ethereum DApp using Web3.js and Truffle. This model solves the Big data copyright issue. To add more value to the model, I have also used the watermarking technique.
